# 装饰器

顾明思义，就是对原对象进行装饰，在不改变原对象的情况下，增加更多的功能。比如，手机(原对象)上贴膜(保护屏幕功能)和加装手机壳(保护手机，防止摔坏)等新功能，未改变原有对象。

装饰器一般形式：

```python
def decorator(func):
    @wraps(func)
    def inner(*args, **kwargs):
        # ...
        return func(*args, **kwargs)
    
    return inner

@decorator
def func():
    pass
```

等价于：

```python
func = decorator(func)
```

*注：inner中的可变参和关键字参数，是用于传递给func函数的参数，通用性较好。当然，也可以直接与func的参数保持一致，但这样装饰器的内部函数就失去通用性。*

模块被**导入**的时候，将被调用装饰动作。

* classmethod装饰器
* staticmethod装饰器
* 一般装饰器
* 带参装饰器
* 多层装饰器
* 装饰器注册

## classmethod装饰

使用`classmethod`修饰后，被装饰的函数foo将成为类的函数方法，可使用cls.foo直接调用，也可以访问类属性。

```python
class Demo(object):
    _a = 10
    def __init__(self):
        pass

    @classmethod
    def foo(cls, num):
        print(f"a = {cls._a}, num = {num}")

if __name__ == "__main__":
    Demo.foo(3)
```

代码输出：

```python
a = 10, num = 3
```

## staticmethod装饰器

使用`staticmethod`修饰后，被装饰的函数foo将成为类的函数方法，可使用cls.foo直接调用，但不能访问类属性。

```python
class Demo(object):
    _a = 10
    def __init__(self):
        pass

    @staticmethod
    def foo(num):
        print(f"num = {num}")

if __name__ == "__main__":
    Demo.foo(3)
```

代码输出：

```python
num = 3
```

## 一般装饰器

```python
def decorator(func):
    print(f"{decorator.__name__} {func.__name__} in")
    def inner(*args, **kwargs):
        print(f"{inner.__name__} in")
        result = func(*args, **kwargs)
        print(f"{inner.__name__} out")

        return result

    print(f"{decorator.__name__} {func.__name__} out")

    return inner

@decorator
def foo1():
    print(f"{foo1.__name__}")

@decorator
def foo2(a):
    print(f"{foo1.__name__} a = {a}")

@decorator
def foo3(a, b):
    print(f"{foo1.__name__} a = {a}, b = {b}")

if __name__ == "__main__":
    print("main")
    foo1()
    foo2(10)
    foo3(10, 20
```

```python

decorator foo1 out   
decorator foo2 in    
decorator foo2 out   
decorator foo3 in    
decorator foo3 out   
main                 
inner in             
inner                
inner out            
inner in             
inner a = 10         
inner out            
inner in             
inner a = 10, b = 20 
inner out            
```

FAQ: 被装饰函数的__name__和__doc__属性被inner覆盖，有何解决之法？

只要在inner上方添加：

```python
@wraps(inner)
```

修改后如下：


## 带参装饰器

装饰器可以带某些参数，如下面统计函数运行耗时的例子。装饰器带参数，势必需要额外嵌套一层。对foo装饰的代码`@elapsed_time_factory("s")`，是先调用`elapsed_time_factory("s")`函数，将返回值(elapsed_time函数)与@符号组成一起，即`@elapsed_time`作为foo的装饰器。此时，与普通的装饰器并无差别。

```python
import time
from functools import wraps

def elapsed_time_factory(arg_unit):
    unit_map = {
        "s":  1e9,
        "ms": 1e6,
        "us": 1e3,
        "ns": 1
    }

    time_unit = unit_map.get(arg_unit, 1)

    def elapsed_time(func):
        @wraps(func)
        def runner(*args, **kwargs):
            start = time.time_ns()
            result = func(*args, **kwargs)
            end = time.time_ns()

            elapsed = (end - start) / time_unit

            print(f"run {func.__name__} elapse time:{elapsed}{arg_unit}")
            
            return result
        return runner
    return elapsed_time 

@elapsed_time_factory("s")
def foo(sleep_time):
    time.sleep(sleep_time)

@elapsed_time_factory("ms")
def demo(sleep_time):
    time.sleep(sleep_time)

if __name__ == "__main__":
    foo(3) 
    demo(0.3)
```

代码输出：

```python
run foo elapse time:3.0001716s
run demo elapse time:300.0171ms
```

## 多层装饰器

一般形式：

```python
@decorator1
@decorator2
def foo():
    pass
```

等效于：

```python
foo = decorator2(foo)
foo = decorator1(foo)
```

装饰顺序与调用先后顺序，呈现**栈的性质(先入后出)**。

```python
def decorator1(func):
    print(decorator1.__name__, "in")
    def inner():
       print("decorator1 inner in") 
       func()
       print("decorator1 inner out") 

    print(decorator1.__name__, "out")

    return inner

def decorator2(func):
    print(decorator2.__name__, "in")
    def inner():
       print("decorator2 inner in") 
       func()
       print("decorator2 inner out") 

    print(decorator2.__name__, "out")

    return inner

@decorator1
@decorator2
def foo():
    print("run foo")

if __name__ == "__main__":
    print("-" * 20)
    foo()
```

代码输出：

```python
decorator2 in
decorator2 out
decorator1 in
decorator1 out
--------------------
decorator1 inner in
decorator2 inner in
run foo
decorator2 inner out
decorator1 inner out
```
